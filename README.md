# Cutler Equity Research Workbench

Cutler Research AI is an internal Streamlit workbench for creating equity research package workspaces, collecting public documents, uploading authorized licensed materials, and reviewing research package coverage.

## Current Status

Implemented through Phase 3:

- Phase 1 package setup, SQLite persistence, dashboard, validation, shared dark UI, and multi-page navigation.
- Phase 2 SEC company resolution, SEC filing preview/download, investor-relations PDF discovery, document metadata, hashes, duplicate prevention, and collection history.
- Phase 3 secure manual licensed-file uploads, lightweight classification suggestions, analyst category correction, upload history, audit events, ZIP inspection, document inventory editing, controlled licensed-file deletion, and security-type-aware research checklist review.

Not implemented yet:

- Package ZIP generation or manifest generation. This begins in Phase 4.
- Document-content analysis, evidence extraction, citations, RAG, embeddings, or LLM calls. Document-content analysis begins in Phase 5.
- Buy/Sell/Hold recommendations, PM approval, DOCX/PDF investment memos, authentication, or cloud deployment.
- Bloomberg, FactSet, Morningstar, or sell-side APIs/scraping.

## Supported Upload Formats

Phase 3 supports manually uploaded:

- PDF
- XLSX
- XLSM
- CSV
- DOCX
- TXT
- ZIP
- PNG
- JPG / JPEG

Uploaded spreadsheets are stored as inert files only. The app does not execute macros, formulas, scripts, embedded files, vendor add-ins, or ZIP contents.

## Licensed-Document Workflow

1. Create or select a package.
2. Open `Document Collection`.
3. Choose a licensed source type such as Bloomberg, sell-side, credit research, Morningstar, FactSet, transcripts, industry research, financial models, company materials, or other.
4. Drag and drop one or more files.
5. Review validation results, suggested category, and confidence.
6. Correct category, title, dates, source institution, and notes as needed.
7. Confirm the authorization acknowledgement:

```text
I confirm that these files are authorized for internal use and that their storage complies with Cutler Capital's vendor entitlements.
```

8. Upload accepted files.
9. Review package coverage and inventory on `Package Review`.

The acknowledgement is an internal workflow control, not a substitute for legal/vendor entitlement enforcement.

## Document Categories And Classification

Categories are centralized in `app/services/taxonomy.py` and include company filings, earnings materials, Bloomberg exports, FactSet, Morningstar, sell-side research, credit research, rating agency research, activist/short-seller research, models, debt/convertible analysis, internal notes, and other.

Classification is a suggestion only. It uses filename, extension, selected source, and simple keyword rules such as:

- `Bloomberg` + `DES` -> Bloomberg DES
- `ANR` -> Bloomberg ANR
- `DRSK` -> Bloomberg DRSK
- `10-K` -> Annual Filing
- `10-Q` -> Quarterly Filing
- `transcript` -> Earnings Transcript
- `initiation` -> Sell-Side Initiation Report
- `Moody`, `S&P`, `Fitch` -> Rating Agency Research
- `credit` -> Credit Research
- `model` in spreadsheet filename -> Financial Model
- `short report` -> Short-Seller Research

The app does not claim to understand document contents in Phase 3.

## Research Checklist

The checklist is generated by security type:

- Common Equity
- Convertible Security
- Credit / Debt
- Other

Statuses include `AVAILABLE`, `MISSING`, `NOT_AVAILABLE`, `NOT_APPLICABLE`, `NEEDS_REVIEW`, and `STALE`. Documents automatically mark matching items available by category, but analysts must manually mark items not available or not applicable and can add notes. Overrides are preserved separately from automatic status.

## ZIP Upload Safety

ZIP files can be uploaded and inspected, but Phase 3 stores ZIPs archive-only. Automatic extraction is not enabled. The inspection view rejects or flags unsafe entries such as absolute paths, `../` traversal, unsupported file types, excessive entry counts, excessive uncompressed size, and suspicious compression ratios.

## Duplicate Detection And Deletion

Uploads compute SHA-256 before final storage. Duplicate hashes in the same package are recorded as duplicate and are not copied again. Controlled deletion is available for manually uploaded licensed files only after explicit confirmation. Deletions are constrained to managed package directories and create audit events.

## Security Controls

- Extension whitelist
- Practical file signature checks
- Empty-file rejection
- Per-file and batch-size limits
- Atomic writes
- Filename sanitization and path traversal prevention
- ZIP-slip and ZIP-bomb protections
- SHA-256 hashing
- Duplicate detection
- No macro execution
- No formula refresh
- No embedded file execution
- No external model submission
- No cloud upload
- Safe error messages and audit logging

No antivirus scanning is claimed.

## SEC User-Agent Configuration

SEC requests require a descriptive user-agent:

```powershell
$env:SEC_USER_AGENT = "Cutler Capital Research Workbench research-team@your-domain.com"
$env:SEC_REQUEST_DELAY_SECONDS = "0.2"
$env:SEC_CACHE_HOURS = "24"
```

Do not leave `SEC_USER_AGENT` set to the placeholder in `.env.example`.

## Setup On Windows PowerShell

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

Run tests:

```powershell
pytest
python -m compileall app tests
```

Launch the app:

```powershell
python -m streamlit run app\Home.py --server.port 8505
```

Or use the launcher from any directory:

```powershell
.\scripts\run_app.ps1
```

## Known Limitations

Phase 3 does not parse document content for investment analysis. It stores files, metadata, classification suggestions, and checklist coverage only. Public collection remains conservative; JavaScript-heavy, blocked, authenticated, malformed, or paywalled sites may require manual handling.
